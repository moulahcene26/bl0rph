# abdelmounaim.xyz

A plain static blog. Authoring is just markdown; a tiny zero-dependency
`build.js` prerenders each post into a static page with real `<head>`
metadata (so shared links get title + image previews) and emits
`sitemap.xml` / `robots.txt` for SEO. Deployed to GitHub Pages from
`main` (see `.github/workflows/static.yml`), served at
[abdelmounaim.xyz](https://abdelmounaim.xyz).

## Files

| file           | role                                                          |
|----------------|--------------------------------------------------------------|
| `index.html`   | landing page: about + the blog index (built from `posts.js`) |
| `posts.js`     | the ordered list of entries, the only thing you edit to publish |
| `build.js`     | prerenders posts, bakes OG/Twitter/JSON-LD meta, writes sitemap/robots |
| `md.js`        | tiny hand-written markdown to HTML renderer                  |
| `theme.js`     | dark/light toggle (defaults to OS, remembers your choice)    |
| `style.css`    | all styling; light + dark, OS default overridable by toggle  |
| `posts/`       | one folder per entry, holding `index.md` and its images      |
| `post.html`    | legacy redirect: `/post.html?p=<slug>` → `/posts/<slug>/`    |
| `404.html`     | not-found page                                               |

Generated by `build.js` (git-ignored, rebuilt on every deploy):
`posts/<slug>/index.html`, `sitemap.xml`, `robots.txt`.

## Publish a new entry

1. Create `posts/<slug>/index.md` (slug = lowercase letters, digits, hyphens).
   Put any images (`.png`, `.jpg`, `.svg`, `.gif`) in that same folder and
   reference them by filename: `![alt](screenshot.png)`. An image alone on
   its own line gets its alt text rendered as a caption.
   - Control image width with a trailing `{...}`: a bare number is a
     fraction of the text column (`1` = full), or use `%`/`px`:
     `![alt](shot.png){width=0.6}`, `![alt](shot.png){width=120%}`.
2. Add an object to the **top** of the `POSTS` array in `posts.js`:

   ```js
   {
     slug: "my-first-finding",
     title: "IDOR in the billing export",
     date: "2026-08-01",
     tags: ["idor", "web"],
     summary: "One line shown on the index and used as the share description.",
     thumb: "posts/my-first-finding/cover.png", // optional; index thumbnail + share image
   },
   ```
3. Commit and push to `main`. The Pages workflow runs `node build.js` and
   redeploys automatically.

### Share images

`thumb` doubles as the Open Graph / Twitter image for that post. For the
best-looking previews use a raster (`.png`/`.jpg`) around **1200×630**.
SVG works on the site but some platforms won't render it as a share image.

See `posts/syntax-reference/` for every markdown construct the renderer
supports; copy that folder as a starting template.

## Preview locally

```sh
node build.js                 # generate the post pages + sitemap
python3 -m http.server 8099   # then open http://127.0.0.1:8099/
```

Serve over HTTP (not `file://`) so the pages resolve correctly. Re-run
`node build.js` after editing a post or `posts.js` — or run
`node build.js --watch` to rebuild automatically on every save.
Requires Node (any recent version); no packages to install.
